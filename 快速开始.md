# FPK 快速开始指南

## 创建你的第一个 Native 应用

### 1. 创建应用

创建 Node.js Web 服务应用：

```
create_application(
  name="hello-world",
  type="native",
  description="一个简单的 Node.js Web 服务"
)
```

### 2. 查看应用结构

```
open_application("hello-world")
```

输出：
```json
{
  "name": "hello-world",
  "path": ".../projects/hello-world",
  "type": "native",
  "manifest": {
    "appname": "hello-world",
    "name": "hello-world",
    "version": "1.0.0",
    "type": "native",
    "author": "snltty",
    "description": "一个简单的 Node.js Web 服务"
  },
  "fileCount": 0,
  "files": []
}
```

### 3. 创建服务端程序

创建 `app/server/package.json`：

```
write_file(
  app_name="hello-world",
  file_path="app/server/package.json",
  content='{\n  "name": "hello-world",\n  "version": "1.0.0",\n  "description": "Hello World 应用",\n  "main": "server.js",\n  "scripts": {\n    "start": "node server.js"\n  }\n}'
)
```

创建 `app/server/server.js`：

```
write_file(
  app_name="hello-world",
  file_path="app/server/server.js",
  content='const http = require("http");\n\nconst hostname = "0.0.0.0";\nconst port = process.env.PORT || 3000;\n\nconst server = http.createServer((req, res) => {\n  res.statusCode = 200;\n  res.setHeader("Content-Type", "text/plain");\n  res.end("Hello World\\n");\n});\n\nserver.listen(port, hostname, () => {\n  console.log(`Server running at http://${hostname}:${port}/`);\n});'
)
```

### 4. 更新 Manifest 配置

```
edit_manifest(
  app_name="hello-world",
  updates={
    "entry": "server.js",
    "port": "3000",
    "auto_start": "true"
  }
)
```

### 5. 验证应用

```
validate_application("hello-world")
```

输出：
```json
{
  "valid": true,
  "errors": [],
  "warnings": []
}
```

### 6. 打包应用

```
build_fpk("hello-world")
```

输出：
```json
{
  "success": true,
  "duration": "1234ms",
  "fpk": {
    "path": ".../projects/hello-world/hello-world.fpk",
    "filename": "hello-world.fpk",
    "size": 12345,
    "sizeHuman": "12.06 KB"
  }
}
```

## 创建你的第一个 Docker 应用

### 1. 创建应用

```
create_application(
  name="nginx-app",
  type="docker",
  description="Nginx Web 服务器"
)
```

### 2. 配置 Docker Compose

```
edit_docker_compose(
  app_name="nginx-app",
  content="version: '3.8'\nservices:\n  nginx:\n    image: nginx:alpine\n    ports:\n      - '80:80'\n    volumes:\n      - ./html:/usr/share/nginx/html:ro\n    restart: unless-stopped"
)
```

### 3. 更新 Manifest

```
edit_manifest(
  app_name="nginx-app",
  updates={
    "port": "80",
    "privileged": "false"
  }
)
```

### 4. 验证并打包

```
validate_application("nginx-app")
build_fpk("nginx-app")
```

## 导入现有 FPK

```
import_fpk(
  fpk_path="C:\\Users\\me\\Downloads\\existing-app.fpk",
  target_name="imported-app"
)

open_application("imported-app")
list_files("imported-app")

# 查看并修改
read_file("imported-app", "app/server/server.js")

# 重新打包
build_fpk("imported-app")
```

## 添加国际化支持

### 1. 查看现有 i18n 文件

```
list_i18n_files("my-app")
```

### 2. 添加翻译

```
edit_i18n(
  app_name="my-app",
  language="zh-CN",
  translations={
    "app.title": "我的应用",
    "app.welcome": "欢迎使用",
    "button.submit": "提交",
    "button.cancel": "取消"
  }
)
```

### 3. 添加英文翻译

```
edit_i18n(
  app_name="my-app",
  language="en-US",
  translations={
    "app.title": "My App",
    "app.welcome": "Welcome",
    "button.submit": "Submit",
    "button.cancel": "Cancel"
  }
)
```

## 常见任务

### 修改端口号

```
edit_manifest(
  app_name="my-app",
  updates={"port": "8080"}
)
```

### 添加环境变量

```
edit_env(
  app_name="my-app",
  env_vars={
    "NODE_ENV": "production",
    "DB_HOST": "localhost",
    "DB_PORT": "5432"
  }
)
```

### 添加静态文件

创建静态页面：

```
write_file(
  app_name="my-app",
  file_path="app/web/index.html",
  content='<!DOCTYPE html>\n<html>\n<head>\n  <title>My App</title>\n</head>\n<body>\n  <h1>Hello World</h1>\n</body>\n</html>'
)
```

### 列出所有文件

```
list_files("my-app", recursive=true)
```

## 故障排除

### 打包失败

1. 运行验证检查：
   ```
   validate_application("my-app")
   ```

2. 查看错误信息并修复

3. 重新打包

### 找不到文件

1. 列出所有文件确认路径：
   ```
   list_files("my-app", recursive=true)
   ```

2. 使用正确的相对路径

### Manifest 格式错误

Manifest 文件使用简单的 `key=value` 格式，每行一个配置：

```
key=value
key2=value2
```

不要使用 YAML 或 JSON 格式。

## 最佳实践

1. **验证先行**：打包前始终运行 `validate_application`
2. **增量编辑**：使用 `edit_file` 而不是 `write_file` 修改现有文件
3. **版本管理**：使用语义化版本号（如 1.0.0）
4. **文档完善**：在 manifest 中添加完整描述
5. **测试运行**：打包后在飞牛系统测试
