# Fnpackup FPK 打包 MCP 服务器设计

## 概述
设计一个 MCP (Model Context Protocol) 服务器，使 AI agent 能够帮助用户创建、编辑和打包 fpk 应用程序。

## FPK 打包工作流程

根据 fnpackup 的功能：
1. **创建应用** - 创建新应用或导入现有 .fpk 文件
2. **进入应用** - 打开应用文件夹
3. **上传/编辑程序** - 原生应用上传程序到 app/server，Docker 应用编辑 app/docker/docker-compose.yaml
4. **编辑配置** - 快速编辑、单独编辑文件或源码编辑
5. **打包应用** - 生成 .fpk 文件

## MCP 工具设计

### 应用管理工具

#### 1. `list_applications`
**描述**: 列出所有应用
**参数**: 无
**返回**: 应用列表（名称、类型、最后修改时间）

#### 2. `create_application`
**描述**: 创建新应用
**参数**:
- `name` (string, required): 应用名称
- `type` (string, required): 应用类型 ("native" | "docker")
- `description` (string, optional): 应用描述

**返回**: 应用路径和基本信息

#### 3. `import_fpk`
**描述**: 导入现有 .fpk 文件
**参数**:
- `fpk_path` (string, required): .fpk 文件路径
- `target_name` (string, optional): 目标应用名称（默认使用原文件名）

**返回**: 导入的应用信息

#### 4. `open_application`
**描述**: 打开应用（进入应用目录）
**参数**:
- `app_name` (string, required): 应用名称

**返回**: 应用目录结构和当前状态

#### 5. `delete_application`
**描述**: 删除应用
**参数**:
- `app_name` (string, required): 应用名称
- `confirm` (boolean, required): 确认删除

**返回**: 删除结果

### 文件编辑工具

#### 6. `list_files`
**描述**: 列出应用目录结构
**参数**:
- `app_name` (string, required): 应用名称
- `path` (string, optional): 相对路径（默认根目录）
- `recursive` (boolean, optional): 是否递归

**返回**: 文件列表

#### 7. `read_file`
**描述**: 读取文件内容
**参数**:
- `app_name` (string, required): 应用名称
- `file_path` (string, required): 相对文件路径

**返回**: 文件内容

#### 8. `write_file`
**描述**: 写入/创建文件
**参数**:
- `app_name` (string, required): 应用名称
- `file_path` (string, required): 相对文件路径
- `content` (string, required): 文件内容
- `encoding` (string, optional): 编码（默认 utf-8）

**返回**: 写入结果

#### 9. `edit_file`
**描述**: 编辑文件（智能编辑）
**参数**:
- `app_name` (string, required): 应用名称
- `file_path` (string, required): 相对文件路径
- `edits` (array, required): 编辑操作列表
  - `old_string`: 要替换的文本
  - `new_string`: 新文本

**返回**: 编辑结果

#### 10. `delete_file`
**描述**: 删除文件或目录
**参数**:
- `app_name` (string, required): 应用名称
- `file_path` (string, required): 相对文件路径
- `confirm` (boolean, required): 确认删除

**返回**: 删除结果

#### 11. `upload_file`
**描述**: 上传文件到应用（原生应用）
**参数**:
- `app_name` (string, required): 应用名称
- `local_path` (string, required): 本地文件路径
- `target_path` (string, required): 应用内目标路径

**返回**: 上传结果

### 配置编辑工具

#### 12. `edit_manifest`
**描述**: 编辑 manifest 配置文件
**参数**:
- `app_name` (string, required): 应用名称
- `updates` (object, required): 要更新的字段

**返回**: 更新后的 manifest 内容

#### 13. `edit_env`
**描述**: 编辑环境变量配置
**参数**:
- `app_name` (string, required): 应用名称
- `env_vars` (object, required): 环境变量字典

**返回**: 更新结果

#### 14. `edit_docker_compose`
**描述**: 编辑 Docker Compose 配置
**参数**:
- `app_name` (string, required): 应用名称
- `content` (string, required): docker-compose.yaml 内容

**返回**: 更新结果

### 打包工具

#### 15. `build_fpk`
**描述**: 打包应用为 .fpk 文件
**参数**:
- `app_name` (string, required): 应用名称
- `output_path` (string, optional): 输出路径（默认应用目录）

**返回**:
- `fpk_path`: 生成的 .fpk 文件路径
- `size`: 文件大小
- `duration`: 打包耗时

#### 16. `build_with_options`
**描述**: 使用自定义选项打包
**参数**:
- `app_name` (string, required): 应用名称
- `options` (object, optional):
  - `compression_level`: 压缩级别 (0-9)
  - `include_source`: 是否包含源码
  - `sign`: 是否签名

**返回**: 打包结果

### 国际化工具

#### 17. `list_i18n_files`
**描述**: 列出国际化文件
**参数**:
- `app_name` (string, required): 应用名称

**返回**: 国际化文件列表

#### 18. `edit_i18n`
**描述**: 编辑国际化内容
**参数**:
- `app_name` (string, required): 应用名称
- `language` (string, required): 语言代码（如 en-US, zh-CN）
- `translations` (object, required): 翻译字典

**返回**: 更新结果

### 验证工具

#### 19. `validate_application`
**描述**: 验证应用配置
**参数**:
- `app_name` (string, required): 应用名称

**返回**:
- `valid`: 是否有效
- `errors`: 错误列表
- `warnings`: 警告列表

#### 20. `preview_fpk`
**描述**: 预览 fpk 内容（不解压）
**参数**:
- `app_name` (string, required): 应用名称
- `or_fpk_path` (string, optional): 或直接指定 .fpk 文件路径

**返回**: fpk 内容清单

│   ├── index.ts              # MCP 服务器入口
│   ├── tools/               # 工具实现
│   │   ├── app-manager.ts   # 应用管理

- 临时文件处理

## 工作流程示例

### 示例 1: 创建原生应用
```
1. create_application("my-app", "native")
2. upload_file("my-app", "/local/server.js", "app/server/server.js")
3. edit_manifest("my-app", {...})
4. build_fpk("my-app")
→ 生成 my-app.fpk
```

### 示例 2: 创建 Docker 应用
```
1. create_application("my-docker-app", "docker")
2. write_file("my-docker-app", "app/docker/docker-compose.yaml", "...")
3. edit_manifest("my-docker-app", {...})
4. build_fpk("my-docker-app")
→ 生成 my-docker-app.fpk
```

### 示例 3: 导入并修改
```
1. import_fpk("/path/to/existing.fpk", "imported-app")
2. open_application("imported-app")
3. edit_file("imported-app", "app/server/config.js", [...edits])
4. build_fpk("imported-app")
→ 生成修改后的 .fpk
```

## 安全考虑

1. **路径验证**: 所有文件操作限制在项目目录内
2. **命令执行**: 不执行任意 shell 命令，只调用 fnpack
3. **用户确认**: 删除、覆盖等危险操作需要用户确认
4. **资源限制**: 限制文件大小和操作超时
