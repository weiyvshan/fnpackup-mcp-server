# fnpackup-mcp-server 配置示例

## Claude Code 配置

编辑 Claude Code 配置文件 `~/.claude/mcp.json`：

```json
{
  "mcpServers": {
    "fnpackup": {
      "command": "fnpackup-mcp-server",
      "env": {
        "PROJECTS_DIR": "/path/to/fnpackup/projects",
        "FNPACKUP_SERVICE_URL": "http://localhost:1069"
      }
    }
  }
}
```

**注意**: 请根据你的操作系统替换路径：
- **Windows**: `C:\\path\\to\\fnpackup\\projects`
- **macOS/Linux**: `/path/to/fnpackup/projects`

## Claude Desktop 配置

**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "fnpackup": {
      "command": "fnpackup-mcp-server",
      "env": {
        "PROJECTS_DIR": "/path/to/fnpackup/projects",
        "FNPACKUP_SERVICE_URL": "http://localhost:1069"
      }
    }
  }
}
```

**注意**: 请根据你的操作系统替换路径：
- **Windows**: `C:\\path\\to\\fnpackup\\projects`
- **macOS/Linux**: `/path/to/fnpackup/projects`

- **PROJECTS_DIR** (必需): fnpackup 服务的项目目录路径
  - 默认值: `../projects/`（相对于 MCP 服务器的位置）
  - 示例（Windows）: `C:\\path\\to\\fnpackup\\projects`
  - 示例（macOS/Linux）: `/path/to/fnpackup/projects`

- **FNPACKUP_SERVICE_URL** (可选): fnpackup 服务的 API 地址
  - 默认值: `http://localhost:1069`
  - 如果 fnpackup 服务运行在其他地址，需要设置此变量

## 前置要求

1. **启动 fnpackup 服务**: 确保 fnpackup Web 服务已启动并运行在配置的地址
   ```bash
   # 在 fnpackup 目录下
   cd fnpackup
   dotnet run
   ```

2. **确保项目目录存在**: MCP 服务器会尝试创建 PROJECTS_DIR，但需要确保有写入权限

3. **Node.js 版本**: 需要 Node.js 18 或更高版本（支持原生 fetch API）
