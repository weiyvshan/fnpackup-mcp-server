# 安装和配置指南

## 系统要求

- Node.js 18 或更高版本
- fnpack 工具（fnpack.exe）
- Claude Desktop（或支持 MCP 的其他客户端）

## 安装步骤

### 1. 安装 Node.js 依赖

```bash
cd fnpackup-mcp-server
npm install
```

### 2. 构建项目

```bash
npm run build
```

这将生成 `dist/` 目录下的编译文件。

### 3. 配置 Claude Desktop

#### Windows

1. 打开 Claude Desktop
2. 进入设置（Settings）
3. 点击 "Developer" -> "Edit Config"
4. 在配置文件中添加：

```json
{
  "mcpServers": {
    "fnpackup": {
      "command": "node",
      "args": ["/path/to/fnpackup-mcp-server/dist/index.js"],
      "env": {
        "PROJECTS_DIR": "/path/to/fnpackup/projects"
      }
    }
  }
}
```

**注意**: 根据你的实际安装路径修改 `args` 和 `PROJECTS_DIR`。

5. 重启 Claude Desktop

#### macOS

配置文件位置：`~/Library/Application Support/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "fnpackup": {
      "command": "node",
      "args": ["/path/to/fnpackup-mcp-server/dist/index.js"],
      "env": {
        "PROJECTS_DIR": "/path/to/fnpackup/projects"
      }
    }
  }
}
```

#### Linux

配置文件位置：`~/.config/Claude/claude_desktop_config.json`

```json
{
  "mcpServers": {
    "fnpackup": {
      "command": "node",
      "args": ["/path/to/fnpackup-mcp-server/dist/index.js"],
      "env": {
        "PROJECTS_DIR": "/path/to/fnpackup/projects"
      }
    }
  }
}
```

### 4. 验证安装

在 Claude Desktop 中启动新对话，输入：

```
列出所有可用的 MCP 工具
```

或

```
列出所有 fpk 应用
```

如果看到工具列表或应用列表，说明配置成功。

## 环境变量

### PROJECTS_DIR

指定 fnpackup 的 projects 目录路径。

**默认值**: `../projects/`（相对于 MCP 服务器）

**示例**:

```json
{
  "mcpServers": {
    "fpk-packager": {
      "command": "node",
      "args": ["dist/index.js"],
      "env": {
        "PROJECTS_DIR": "/path/to/your/fnpackup/projects"
      }
    }
  }
}
```

## 项目结构说明

```
fnpackup/                          # 主项目目录
├── fnpackup/                     # 后端
│   ├── projects/                 # ← 应用存储目录（PROJECTS_DIR）
│   ├── statics/                  # 静态托管内容
│   └── fnpack.exe                # 打包工具
└── fnpackup.web/                 # 前端

fnpackup-mcp-server/                 # MCP 服务器
├── dist/                         # 编译输出
└── src/                          # 源代码
```

## 故障排除

### Claude Desktop 无法启动

1. 检查配置文件语法是否正确（JSON 格式）
2. 确认 Node.js 已安装：`node --version`
3. 确认构建完成：`ls dist/index.js`
4. 查看 Claude Desktop 日志：

   **Windows**: `%APPDATA%\Claude\logs`
   **macOS**: `~/Library/Logs/Claude`

### 找不到 fnpack 工具

MCP 服务器会自动查找：
1. `../fnpack.exe`（相对于 MCP 服务器目录）
2. 系统 PATH 中的 `fnpack.exe`

如果找不到，确保 fnpack.exe 在正确位置或添加到 PATH。

### 权限错误

确保 Claude Desktop 有权限：
- 读取项目目录
- 写入 projects 目录
- 执行 fnpack.exe

**Windows**: 右键 Claude Desktop -> "以管理员身份运行"

### MCP 工具未显示

1. 确认已重启 Claude Desktop
2. 检查配置文件路径正确
3. 查看 Claude Desktop 日志中的错误信息

### 打包失败

1. 先运行验证：
   ```
   validate_application("app-name")
   ```

2. 检查常见问题：
   - Manifest 配置错误
   - 缺少必需文件
   - 文件路径错误

## 调试模式

### 查看 MCP 服务器日志

修改 `src/index.ts`，添加日志输出：

```typescript
console.error = (...args) => {
  fs.appendFileSync('/tmp/mcp-debug.log', args.join(' ') + '\n');
};
```

### 测试 MCP 服务器

使用 MCP Inspector：

```bash
npx @modelcontextprotocol/inspector node dist/index.js
```

## 更新

当 MCP 服务器更新后：

```bash
cd fnpackup-mcp-server
git pull
npm install
npm run build
# 重启 Claude Desktop
```

## 卸载

1. 从 Claude Desktop 配置中移除 MCP 服务器配置
2. 删除 `fnpackup-mcp-server` 目录
