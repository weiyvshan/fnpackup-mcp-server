## 🔧 修复详情

### 问题 1: build_fpk 调用本地 CLI（严重）

#### 问题描述
- **文件**: `src/tools/builder.ts`
- **行为**: `buildFpk()` 方法直接 `spawn("fnpack build ...")`
- **影响**:
  - 绕过 fnpackup 服务层
  - 产物路径不可控
  - Hermes 环境下状态管理失败

#### 修复方案
**改为通过 HTTP 调用 fnpackup 服务 API**:

```typescript
// 修改前：直接调用 fnpack CLI
const fnpackPath = this.findFnpack();
const args = ["build", appName];
const result = await this.executeCommand(fnpackPath, args, { cwd });

// 修改后：调用 fnpackup 服务 API
const response = await fetch(`${this.fnpackupServiceUrl}/project/pack?name=${appName}`, {
  method: "POST",
  headers: { "Accept": "application/json" },
});
const data = await response.json();
```

**移除的代码**:
- ❌ `findFnpack()` - 查找本地 fnpack
- ❌ `executeCommand()` - 执行 shell 命令
- ❌ `findGeneratedFpk()` - 搜索 .fpk 文件
- ❌ `findFpkInTempDir()` - 搜索临时目录
- ❌ `child_process` 依赖
- ❌ `util.promisify` 依赖

**新增的代码**:
- ✅ `callFnpackupService()` - HTTP API 调用
- ✅ `fnpackupServiceUrl` - 服务地址配置
- ✅ 清晰的连接失败错误提示

### 问题 2: PROJECTS_DIR 环境变量不生效（中等）

#### 问题描述
- **文件**: `src/tools/app-manager.ts`
- **行为**: 文档说明 `PROJECTS_DIR` 环境变量，但代码从未读取
- **影响**: 用户无法自定义项目目录

#### 修复方案
在 `AppManager` constructor 中添加环境变量读取：

```typescript
constructor() {
  // 优先使用环境变量 PROJECTS_DIR，否则使用默认路径
  const envProjectsDir = process.env.PROJECTS_DIR;
  if (envProjectsDir) {
    this.projectsDir = path.resolve(envProjectsDir);
  } else {
    // 默认指向 fnpackup 的 projects 目录
    this.projectsDir = path.resolve(process.cwd(), "..", "projects");
  }
}
```

## 📊 API 调用流程

### 修改前的流程（错误）

```
MCP Server
  └─> spawn("fnpack build appname")
       └─> 本地 fnpack CLI
            └─> 生成 .fpk（位置不确定）
                 └─> MCP Server 在文件系统搜索
                      └─> ❌ 经常找不到
```

### 修改后的流程（正确）

```
MCP Server
  └─> POST http://localhost:1069/project/pack?name=appname
       └─> fnpackup Service
            └─> spawn("fnpack build appname")
                 └─> 生成 .fpk 到 ./projects/appname/
            └─> 返回 { FileName: "appname.fpk", Msg: "Packing successfully" }
       └─> MCP Server 从响应获取文件名
            └─> ✅ 直接从应用目录读取
```

## ⚠️ 重要说明

### fnpackup 服务认证

**fnpackup 服务不需要任何认证**。它是一个本地/局域网服务，设计用于：
- 本地开发环境
- 局域网内信任网络
- 配合 fnpackup Web UI 使用

**MCP server 不需要**：
- ❌ API Key
- ❌ Token
- ❌ 用户名/密码
- ❌ 任何其他认证凭证

**只需确保**：
- ✅ fnpackup 服务已启动
- ✅ 网络可达（默认 `http://localhost:1069`）
- ✅ PROJECTS_DIR 配置正确

### 端口说明

fnpackup 服务使用两个端口：
- **HTTP**: `http://localhost:1069`（推荐 MCP server 使用）
- **HTTPS**: `https://localhost:10699`（自签名证书）

MCP server 默认使用 HTTP，避免 HTTPS 自签名证书问题。
